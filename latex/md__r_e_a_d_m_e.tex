Afin de valider la troisième année de Licence Informatique, je dois réaliser un projet de synthèse sur une durée d\textquotesingle{}environ 4 mois. Ce projet a pour but de mettre en oeuvre l\textquotesingle{}ensemble des notions découvertes durant ces trois années universitaires. Les notions concernées comprennent celles vues dans les différents projets / TD / TP de développement et de programmation (les design pattern, les langages, etc). Elles comprennent aussi les notions d\textquotesingle{}organisations acquises en dehors des cours universitaires et le stage.

Ce projet est initialement prévu pour être réalisé avec deux personnes. Cependant, la quantité de notions que propose ce projet étant assez conséquente, j\textquotesingle{}ai choisi de réaliser ce projet seul afin de mettre à profit la totalité de mes compétences.

L\textquotesingle{}application à créer doit permettre de construire et dessiner des formes et des groupes de formes à l\textquotesingle{}aide d\textquotesingle{}une application client/serveur à l\textquotesingle{}aide de deux technologies\+: C++ ({\itshape pour la partie client}) et Java ({\itshape pour la partie serveur}). Le fait d\textquotesingle{}utiliser Java ET C++ dans une même application permet de profiter des performances de calcule de C++ et les performances de gestion d\textquotesingle{}interface graphique de Java.

Le but de ce projet est d\textquotesingle{}utiliser toutes les compétences acquises durant ces 3 années de Licence Informatique ({\itshape dans mon cas, mes 2 ans de DUT Informatique puis ma 3ème années de Licence Informatique}) et notamment les compétences suivantes \+:
\begin{DoxyItemize}
\item Programmtion orientée objet
\item Classes et classes abstraites
\item Design Pattern
\item Réseau ({\itshape protocole TCP/\+IP, avec l\textquotesingle{}utilisation des sockets})
\item Système {\itshape ou programmation système} ({\itshape multi-\/threading})
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{Design Pattern}\label{md__r_e_a_d_m_e_autotoc_md2}
Durant ces trois années, j\textquotesingle{}ai pu apprendre plusieurs Design Pattern \+:
\begin{DoxyItemize}
\item MVC
\item Factory
\item {\itshape Singleton}
\item {\itshape Chain of responsability}
\item {\itshape Visitor}
\end{DoxyItemize}

Les trois dernier Design Pattern ont été acquis durant le développement de cette application.\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{Multi-\/threading}\label{md__r_e_a_d_m_e_autotoc_md3}
Le serveur doit pouvoir être géré en multi-\/threading. Dans le cadre de ce projet, il est inutile de géré les exclusions mutuelles car aucune données ne sera aménée à être modifiée simultanément sur plusieurs processus.
